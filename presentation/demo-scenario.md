# 시연 시나리오 (Final Demo Script · v2)

> **컨텍스트**: 13주차(v40)에서 *설계*는 다뤘다. 오늘은 *완성된 제품*과 *라이브 시연*에 집중한다.
> **변경 반영(v2)**: 통화 명칭 **콘** / 관리자 탭 재정렬 + **스토어 관리** 신규 / 유찰 모달 *EVENT·REOPEN* 두 모드 / 경매 목록 3-view·연도 필터.
> **원칙**: 한 화면 = 한 기능. *왜 만들었나*는 잘라내고 *무엇이 동작하는가*만.

## 3. 액트 1 · 사용자가 1일권을 산다 (5분)

**[목표]**: 입찰 → OUTBID 자동환불 → 낙찰 → AUCTION 연차 자동 부여까지 한 흐름.

### 3-1. 로그인 + 대시보드 (40초)

**[조작]** 시크릿창 A에서 `exam001@exam.com` / `1234`.

**[화면 짚기]**
- 우상단 잔액 **100,000 콘**
- 내 연차 위젯: **REGULAR 15일** 파란 막대
- "내 복지 콘" / "에스크로 N콘" 표기 — 통화 단위 콘

> "이메일이 `@exam.com`이라 로컬 인증으로 자동 라우팅됐습니다."

### 3-2. 경매 목록 → 상세 (1분)

**[조작]** 좌측 메뉴 → 경매 목록.

**[화면 짚기]**
- 상단 **연도 셀렉터** (LeavePool 익년 매물 격리)
- **3-view 토글**: Grid / Row / Timeline
- **마감 탭**에 낙찰가/낙찰자/마감시각 + **에스크로 적립 합계** 노출

**[조작]** Grid 뷰에서 1일권 매물 클릭 → 상세 진입 → **30,100 콘 입찰**.

**[관찰 포인트]**
- 시작가 **30,000 콘** 고정 정책
- 입찰 직후 우상단 잔액 **−30,100** 즉시 반영

> "입찰 즉시 지갑 차감 + 에스크로 입금 + 원장 BID 한 줄 INSERT가 **단일 트랜잭션**으로 묶입니다."

### 3-3. 다른 사용자가 덮어쓰기 → 자동 환불 (1분 30초)

**[조작]** 시크릿창 B에서 `exam002@exam.com` 로그인 → 같은 매물 → 30,200 콘 입찰.

**[관찰 포인트]**
- 창 B: 잔액 −30,200, 토스트 "최고가 입찰"
- 창 A 새로고침: 잔액 **+30,100 복구** + 알림 종에 **"덮어쓰기 당했습니다 (OUTBID)"**

> "패배자에게 자동 환불, 알림 옵저버가 토스트까지 띄웠습니다."

### 3-4. 즉시 마감 → 낙찰 → AUCTION 연차 (1분 30초)

**[전환]** 관리자 탭에서 `super@admin.local` 로그인 → **`/admin/auctions`** → 그 매물 **즉시 마감**.

**[관찰 포인트]**
- 창 B: 토스트 **"낙찰! AUCTION 1일 부여됨"**
- 대시보드: **AUCTION 1일 노란 막대** 추가
- 잔액은 그대로 (30,200 콘은 이미 에스크로 확정)

> "AUCTION 연차는 즉시 잔고에 들어옵니다. 외부 그룹웨어 호출 없는 단일 DB 트랜잭션."

## 5. 액트 3 · 관리자 운영 한 바퀴 (6분)

**[목표]**: 새 탭 순서(사람→매물→상품→신청→원장)를 따라 핵심 5개 화면을 한 흐름으로.

### 5-1. 운영 대시보드 (50초)

**[조작]** `super@admin.local` → **`/admin/ops`**.

**[화면 짚기]**
- KPI 5칸 — 에스크로 잔액(콘) · 진행 / 오픈예정 / 유찰 / 오늘 낙찰
- "정산 데이터 내보내기" 카드 → Excel(.xlsx) / Markdown / JSON, 3종 항목 선택

> "KPI는 실시간, 내보내기는 급여팀 전달용으로 Excel·MD·JSON 모두."

### 5-2. 유찰 매물 수동 처리 — 2개 모드 (1분 30초) ⭐

**[조작]** "유찰 재고 (수동 처리 대기)" 카드 **클릭**.

**[화면 짚기 — 새 모달]**
- 매물 정보 박스 (ID · 연차일 · 마감)
- 모드 토글: **EVENT 지급** / **REOPEN(재경매)**
- 직원 검색창 (이름·사번·이메일·부서) — EVENT 모드 시
- 시작/마감 시각 입력 — REOPEN 모드 시

**[조작 A — EVENT 모드]** 검색 `exam003` → 클릭 → **EVENT 휴가 지급**.
- 토스트 "exam003에게 EVENT 1일 지급 — 매물 소진" / KPI "유찰 재고" 1 감소

**[조작 B — REOPEN 모드]** 다른 유찰 매물 클릭 → REOPEN 토글 → 시각 입력 → 재경매.
- 토스트 "기존 매물 → 새 매물 ID로 재오픈" / 경매관리에서 새 매물 확인

> "안 팔린 매물은 이벤트 휴가로 변환하거나, 같은 회사의 새 1일권 경매로 재오픈할 수 있습니다."

### 5-3. 스토어 상품 관리 (1분 30초) ⭐ NEW

**[조작]** 탭에서 **「스토어」** 클릭 → **`/admin/store`**.

**[화면 짚기]**
- 상품 표: sku / 이름 / 브랜드 / 가격(콘) / 재고 / 카테고리 / 표시순서 / 활성
- 우측: "상품 추가" 버튼
- 하단: **감사 로그**(CREATE / UPDATE / ACTIVATE / DEACTIVATE / DELETE 누가 언제)

**[조작]** "상품 추가" → SKU `DEMO-001`, 이름 "데모 상품", 가격 5,000, 카테고리 "데모", 표시순서 0 → 저장.
→ 표에 즉시 행 추가 → 감사 로그에 CREATE 한 줄.

**[조작]** 추가한 상품의 "비활성화" 토글 → 감사 로그에 DEACTIVATE.

**[전환]** 사용자창 A `/redemption` 새로고침 → 비활성화한 상품은 **카탈로그에서 사라짐**.

> "관리자가 상품을 직접 CRUD, 모든 액션은 *누가 언제 무엇*까지 감사 로그에 INSERT-only로 남습니다."

### 5-4. 회원 관리 두 탭 (1분)

**[조작]** **`/admin/members`** → 탭 전환 시연.

**[EZPASS 탭]**

**[EXAM 탭]**

> "한 화면, 두 모드. **위임형(ezpass)** 회사와 **독립형(자체관리)** 회사를 같은 코드로."

### 5-5. 충전 처리 (40초, 옵션 — 시간 되면)

## 6. 액트 4 · 12/31의 마법 (5분, 핵심)

**[목표]**: 시드된 **2025년 배당 데모(39건 AWARDED)** 로 풀 수집 + 배당 → 에스크로 0.

### 6-1. 사전 한 줄 (20초)

> "1년간 모인 에스크로가 12/31에 어떻게 *기여한 사람들*에게 흘러가는지 — 시드에 작년 39건 낙찰 데이터를 미리 넣어뒀습니다."

### 6-2. 연말 풀 수집 (1분 30초)

**[조작]** `/admin/ops` → "풀 수집" 버튼.

**[화면 짚기 — dryRun 모달]**
- 기여자별 일수 미리보기
- 익년도 매물 N개 생성 예정 (LP-2026-* 채번 미리보기)

**[조작]** "수집 실행" → 토스트 → `/admin/auctions`에 **LP-2026-* 매물 새로 등장**.

> "올해 안 쓴 일반(REGULAR) 연차가 자동으로 익년도 경매 인벤토리. **연도당 1회만** 실행 — DB UNIQUE 제약."

### 6-3. 연말 배당 정산 (2분 30초)

**[조작]** `/admin/ops` → "연말 배당" 버튼.

**[화면 짚기 — dryRun 미리보기]**
- 표: 사용자별 stake 일수 / 배당 비율(%) / 분배 콘
- user001 30.8%(12일 기여) / user002~010 각 7.7%(3일)
- 합계 = 에스크로 잔액 정확히 일치

**[조작]** "실지급" 클릭 → 확인.

**[관찰 포인트]**
- 토스트 "10명에게 N콘 배당 (에스크로 전액)"
- KPI 에스크로 잔액 → **0**
- `/admin/ledger`에 **DIVIDEND 항목 10건** INSERT

> "에스크로 0. 회사가 자기 돈 한 푼도 쓰지 않고 1년이 닫혔습니다."

### 6-4. 멱등성 보너스 (30초)

**[관찰 포인트]**

### 7-3. 새 회사 추가하려면? (30초, 말로만)

> "`company` 행 하나, 사용자의 `companyId`만 박으면 끝. 도메인 추가하면 인증 라우팅도 자동입니다."

## 9. Q&A 대비 (기능 위주)

| 질문 | 30초 답변 |
|---|---|
| "P 단위가 콘으로 바뀐 이유?" | 학교 프로젝트의 *내부 가상 통화* 정체성을 명확히 — '포인트'는 일반명사라 외부 채널과 헷갈림. **콘**은 우리 시스템 안에서만 유효한 단위명. |
| "동시 입찰?" | SQLite 쓰기락 + 단일 트랜잭션. 후행은 fresh-read에서 currentPrice 재확인 후 재시도. |
| "낙찰 시 외부 콜?" | 현재 `InternalLeaveAdapter`로 로컬 DB만. 외부 그룹웨어 연동 시 Outbox 패턴 — 어댑터 교체. |
| "배당 두 번?" | `dividend_run.year` UNIQUE 멱등. 시연 6-4에서 보여드림. |
| "스토어 카테고리 추가?" | 방금 보신 `/admin/store`에서 행 추가만. 자동으로 사용자 카탈로그에 새 섹션. |
| "회사 데이터 격리?" | JWT companyId가 모든 쿼리에 자동 스코프. ADMIN(super)만 스위처. |
| "테스트?" | 도메인 unit(DB 없음) `npm test` / E2E `npm run test:e2e`. |
| "감사성?" | 원장 INSERT-only, 스토어 상품도 모든 액션 audit 로그 INSERT-only. UPDATE/DELETE 사후 조작 불가. |
