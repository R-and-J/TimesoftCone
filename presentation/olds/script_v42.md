# 시연 시나리오 (Final Demo Script)

> **컨텍스트**: 13주차 발표(슬라이드 v40)에서 **설계는 이미 다뤘다**. 오늘은 *완성된 제품*과 *라이브 시연*에 집중한다.
> **목표**: 25–30분 안에 "직원 화면 → 관리자 화면 → 연말 정산 → 회사 격리" 4코스 풀 디너를 끊김 없이 보여준다.
> **원칙**: 한 화면 = 한 기능. *왜 이렇게 만들었나*는 잘라내고, *무엇이 동작하는가*만 보여준다.

## 2. 구현된 기능 한눈에 (1분)

**[화면: 발표용 별도 슬라이드 1장 — 기능 인벤토리]**

### 사용자 화면 (5종)
- 로그인 (이메일 도메인 자동 분기: `@timesoftcone.com` → ezpass / `@exam.com` → 로컬)
- 대시보드 — 잔액·내 연차(REGULAR/AUCTION/EVENT 3-flag 막대)·진행 중 경매 위젯·알림 종
- 경매 목록/상세 — 폴링 기반 실시간 갱신
- 입찰 — 단일 트랜잭션, 자동 환불(OUTBID 알림 동반)
- 스토어 — 카탈로그(AI 구독·식권·카페·기프티콘) + 교환 신청 + 우상단 **수령 대기** + **📦 보관함** 팝업
- 충전 요청 — 사용자가 보내고 관리자가 승인 (알림 deep-link)

### 관리자 화면 (7종)
- 운영 대시보드 — KPI 5칸(잔액·진행·예정·유찰·오늘 낙찰) + 정산 내보내기 + 풀수집 + 연말 D-day
- 경매 관리 — 채번/일괄 생성/스케줄/즉시 마감/연장/취소
- 회원 관리 — ezpass 동기화(연동회사) + EXAM 로컬 CRUD(독립회사) + 충전 처리
- 교환신청 처리 — KPI 카드 토글 필터 + 승인(쿠폰) / 반려(환불)
- 원장(Ledger) 조회 — actionType·기간 필터
- 알림 센터 — 종 아이콘 + 미확인 카운트
- 풀 분산 정책 — none/daily/weekly/monthly

### 자동화 (스케줄러 5종, 백그라운드)
- 오픈/마감 자동 정산 — 60초 주기
- 점진 발행(`ReleaseInventoryScheduler`) — 풀 매물 분산 오픈
- 연말 배당 / 연말 풀 수집 — 12/31 자동 트리거 (수동도 가능)
- 유찰 재고 영구 삭제 — 익년 1/1

### 가로지르는 것 (cross-cutting)
- 멀티테넌시 — 회사별 데이터 완전 격리, ADMIN만 스위처
- 5종 role 게이팅 — ADMIN / EZPASS_ADMIN / EXAM_ADMIN / EZPASS / EXAM
- 인증 2모드 — ezpass 위임 + 로컬 비번
- 정산 내보내기 — Excel(.xlsx) / Markdown / JSON

> "지금부터 이 중 **굵게 표시된 것들을 라이브로** 보여드립니다." (시간 압박 시 굵은 것만 골라 시연)

## 3. 액트 1 · 사용자가 1일권을 사고 받는다 (5분)

**[목표]**: 입찰 → OUTBID 자동환불 → 낙찰 → AUCTION 연차 자동 부여까지 *한 흐름*.

### 3-1. 로그인 + 대시보드 (40초)

**[조작]** 시크릿창 A에서 `exam001@exam.com` / `1234`.

**[화면 짚기]**
- 우상단 잔액 **100,000 콘**
- 내 연차 위젯: **REGULAR 15일** 파란 막대 + (아직 AUCTION/EVENT 없음)
- 진행 중 경매 위젯

> "이메일 도메인이 `@exam.com`이라 자동으로 로컬 인증으로 라우팅됐습니다."

### 3-2. 경매 상세 + 입찰 (1분)

**[조작]** `/auctions` → 1일권 매물 클릭 → **30,100 입찰**.

**[화면 짚기]**
- 매물 상태 OPEN, 시작가 30,000 콘 고정
- 입찰 직후 우상단 잔액 **−30,100** 즉시 반영

> "입찰 즉시 지갑이 잠기고 원장에 BID 한 줄이 INSERT됩니다."

### 3-3. 다른 사용자가 덮어쓰기 → 자동 환불 (1분 30초)

**[조작]** 시크릿창 B에서 `exam002@exam.com` 로그인 → 같은 매물 → 30,200 입찰.

**[관찰 포인트]**
- 창 B: 잔액 −30,200, 토스트 "최고가 입찰"
- 창 A 새로고침: 잔액 **+30,100 복구** + 알림 종에 **"덮어쓰기 당했습니다 (OUTBID)"** 알림 1건

> "이전 입찰자에게 자동 환불, 알림 옵저버가 토스트까지 띄웠습니다."

### 3-4. 즉시 마감 → 낙찰 → AUCTION 연차 (1분 30초)

**[전환]** 관리자 탭에서 `super@admin.local` 로그인 → `/admin/auctions` → 그 매물 **즉시 마감**.

**[관찰 포인트]**
- 창 B(낙찰자): 토스트 **"낙찰! AUCTION 1일 부여됨"**
- 대시보드 가서: 내 연차 위젯의 **AUCTION 1일 노란 막대**가 추가됨
- 잔액은 그대로 (30,200은 이미 에스크로 확정)

> "AUCTION 연차는 **즉시 잔고에 입금**됩니다. 외부 그룹웨어 호출 없이 단일 트랜잭션으로요."

## 4. 액트 2 · 스토어 → 보관함 (3분)

**[목표]**: 새 UX의 핵심인 우상단 **수령 대기** 패널과 **보관함** 팝업 보여주기.

### 4-1. 교환 신청 (1분)

**[조작]** 사용자창 A `exam001` → `/redemption`.

**[화면 짚기]**
- 우상단 잔액카드 + 「📦 보관함」 버튼
- 카탈로그 4카테고리 (AI 구독·식권·카페·기프티콘)

**[조작]** **카카오톡 선물하기 10,000원권** → 교환 신청 → 신청 확인.

**[관찰 포인트]**
- 잔액 **−10,000** 잠금
- 토스트 "교환 신청 완료 — 관리자 승인 대기"

### 4-2. 관리자 승인 + 쿠폰 발급 (50초)

**[조작]** 관리자창에서 `/admin/redemption` → 「대기」 KPI 카드 자동 강조 → 신청 행에서 **승인** → 쿠폰 텍스트 `KAKAO-DEMO-1234-5678` 입력 → 승인.

**[관찰 포인트]**
- 신청 상태 → **APPROVED**
- KPI 카드: 대기 -1, 승인(수령 대기) +1

> "KPI 카드는 클릭으로 필터, 같은 카드 다시 누르면 전체로 해제됩니다."

### 4-3. 사용자: 우상단 수령 대기 → 수령 완료 (1분 10초)

**[조작]** 사용자창 A 새로고침.

**[관찰 포인트] — 새 UX 강조**
- 우상단 잔액카드 **아래에** 새로 「📦 수령 대기 1건」 카드가 떠 있음
- 카드 안에 쿠폰 코드 + **수령 완료** 버튼
- "수령 완료" 누름 → 카드 사라짐
- 잔액카드 오른쪽 「보관함 (N)」 버튼 클릭 → 모달 → 방금 수령한 쿠폰 **다시 확인 가능**

> "직원이 화면 어디로도 스크롤 안 해도 *수령 대기*를 놓치지 않고, *이미 받은 쿠폰*도 보관함에서 다시 꺼내 쓸 수 있습니다."

## 5. 액트 3 · 관리자 운영 한 바퀴 (5분)

**[목표]**: 운영 콘솔의 핵심 기능 4종 — KPI / **유찰 매물 수동 처리(신규)** / 회원관리 / 정산 내보내기.

### 5-1. 운영 대시보드 (1분)

**[조작]** `super@admin.local` → `/admin/ops`.

**[화면 짚기]**
- KPI 5칸 — 에스크로 잔액 / 진행 중 / 오픈 예정 / 유찰 재고 / 오늘 낙찰
- "정산 데이터 내보내기" 카드 → 클릭 → 모달에서 *Excel/MD/JSON* + 항목 3종 선택 가능
- "연말 풀 수집" 카드
- 우측: 연말 배당 D-day 카운터

> "KPI는 실시간, 내보내기는 급여팀 전달용으로 Excel·MD·JSON 모두 지원합니다."

### 5-2. 유찰 매물 수동 처리 (1분 30초) ⭐ 신규

**[조작]** "유찰 재고 (수동 처리 대기)" 영역에서 카드 **클릭**.

**[화면 짚기 — 새 모달]**
- 매물 정보 박스(ID/연차일수/마감일)
- 직원 검색창 (이름·사번·이메일·부서)
- 활성 직원 리스트(선택 시 강조 + "선택" 핸들)

**[조작]** 검색에 `exam003` → 클릭 → **EVENT 휴가 지급**.

**[관찰 포인트]**
- 토스트 "exam003에게 EVENT 휴가 1일 지급 — 매물 소진"
- KPI "유찰 재고" 1 감소
- exam003으로 로그인 → 내 연차 위젯에 **EVENT 1일** 노란 막대 추가

> "안 팔린 매물도 그냥 두지 않습니다. 이벤트 휴가로 변환해 임의 직원에게 줄 수 있어요. 매물은 즉시 소진(DB에서 제거)됩니다."

### 5-3. 회원 관리 (1분)

**[조작]** `/admin/members`.

**[화면 짚기]**
- 탭: **ezpass 회원 (연동)** / **exam 회원 (독립)**
- ezpass 탭: 회원 추가는 ezpass에서, 우리는 「지금 동기화」와 「ezpass에서 관리 ↗」만
- EXAM 탭: 직접 회원 추가/수정/비활성화 가능

**[조작]** EXAM 탭 → "회원 추가 (exam)" → 임의 직원 한 명 생성.

> "회사 성격에 따라 **위임형(ezpass)** 과 **독립형(자체관리)** 을 한 탭으로 분리. 한 코드베이스가 두 모드를 다 지원합니다."

### 5-4. 충전 처리 (40초, 옵션)

**[조작]** 임의 회원 행의 "콘 관리" → 자유 충전 +50,000 콘, 사유 "데모 보너스".

**[관찰 포인트]**
- 즉시 반영, 원장에 CREDIT_ADMIN 한 줄 INSERT

> "사유는 필수 — 감사 로그가 깨끗하게 남습니다."

## 6. 액트 4 · 12/31의 마법 (5분, 핵심)

**[목표]**: 시드된 **2025년 배당 데모(39건 AWARDED)** 로 풀 수집 + 배당 정산 → 에스크로 0.

### 6-1. 사전 한 줄 (20초)

> "1년 동안 모인 에스크로가 12/31에 어떻게 *기여한 사람들*에게 흘러가는지 — 시드에 작년 39건 낙찰 데이터를 미리 넣어뒀습니다."

### 6-2. 연말 풀 수집 (1분 30초)

**[조작]** `/admin/ops` → "풀 수집" 버튼.

**[화면 짚기 — dryRun 모달]**
- 기여자별 일수 미리보기
- 익년도 매물 N개 생성 예정 (LP-2026-* 채번 미리보기)

**[조작]** "수집 실행" → 토스트 → `/admin/auctions`에 **LP-2026-* 매물 새로 등장**.

> "올해 안 쓴 일반(REGULAR) 연차가 자동으로 익년도 경매 인벤토리가 됩니다. **연도당 1회만** 실행 — DB UNIQUE 제약으로 두 번 실행해도 사고 안 납니다."

### 6-3. 연말 배당 정산 (2분 30초)

**[조작]** `/admin/ops` → "연말 배당" 버튼.

**[화면 짚기 — dryRun 미리보기]**
- 표: 사용자별 stake 일수 / 배당 비율 / 분배 금액
- user001 30.8% (12일 기여) / user002~010 각 7.7% (3일)
- 합계 = 에스크로 잔액 정확히 일치

**[조작]** "실지급" 클릭 → 확인.

**[관찰 포인트]**
- 토스트 "10명에게 N P 배당 (에스크로 전액)"
- KPI 에스크로 잔액 → **0**
- `/admin/ledger` 가서 **DIVIDEND 항목 10건** INSERT 확인 가능

> "에스크로 0. 회사가 자기 돈 한 푼도 쓰지 않고 1년이 닫혔습니다."

### 6-4. 멱등성 보너스 (30초)

**[조작]** "연말 배당" 한 번 더 클릭.

**[관찰 포인트]**
- 409 "이미 정산되었습니다" 응답 → 토스트 에러
- 데이터 무변경

> "같은 해 두 번 정산 못 합니다. 운영 사고 방지."

### 7-1. 회사 스위처 (1분)

**[관찰 포인트]**
- 회원 목록·경매 목록·원장·통계 — 회사 단위로 *완전히 다른 데이터*로 교체

### 7-2. 다른 회사로 직접 로그인 (1분 30초)

**[조작]** 새 시크릿창 → `examadmin@exam.com` 로그인 → `/admin/members`.

**[관찰 포인트]**

**[조작]** `/admin/auctions` → EXAM 회사 매물만 노출.

> "JWT에 `companyId`가 박혀서 모든 쿼리에 자동 스코프됩니다. 회사 외부 데이터를 *조회조차 못 합니다*."

### 7-3. 새 회사 추가하려면? (30초, 말로만)

## 9. Q&A 대비 (기능 위주)

| 질문 | 30초 답변 |
|---|---|
| "동시 입찰이 들어오면?" | SQLite 쓰기락 + 단일 트랜잭션. 두 번째 입찰은 fresh-read에서 currentPrice 재확인 후 재시도. |
| "낙찰 시 외부 시스템 콜은?" | 현재는 `InternalLeaveAdapter`로 로컬 DB만. 외부 그룹웨어 연동 시 Outbox 패턴 — 어댑터 교체만으로 확장. |
| "배당 두 번 누르면?" | `dividend_run.year` UNIQUE 제약으로 멱등. 두 번째는 409. *방금 보여드린 시연 6-4*. |
| "스토어 카테고리 추가?" | DB `redemption_item` 행 추가만. 자동으로 화면에 새 카테고리 섹션 생성. |
| "왜 SQLite?" | 학교 프로젝트 스코프에서 서버 의존성 0. 실서비스는 PostgreSQL 교체 — Prisma 한 줄. |
| "한 회사가 다른 회사 데이터 보려면?" | 못 봅니다. JWT companyId가 모든 쿼리에 자동 스코프. ADMIN(super)만 스위처로 전환 가능. |
| "테스트?" | 도메인 unit (DB 없음) `npm test` / E2E `npm run test:e2e`. |
| "기능 더 있는데 시간 부족했나요?" | 큰 기능은 다 들어갔습니다. 12/31 자동 배당, 점진 발행, 자동 마감 정산은 시연에선 *수동 트리거*로 보여드렸지만 *백그라운드 스케줄러*가 같은 use case를 호출합니다. |

## 10. 안전장치 — 시연 실패 시 우회

| 실패 | 대안 |
|---|---|
| 백엔드 죽음 | `presentation/img/` 백업 캡쳐 PNG 슬라이드 사용 |
| 즉시 마감 안 됨 | 이미 AWARDED 상태인 매물로 "이 화면이 결과입니다" 화면 캡쳐 사용 |
| 배당 미리보기 비어 보임 | "이미 정산됐어요" 409로 전환 → *멱등성 시연*으로 활용 |
| 멀티테넌트 안 보임 | 회사 격리 ER 슬라이드 1장(`companyId` 컬럼 강조) |
| 시간 초과 | 액트 5(멀티테넌트)를 30초 요약. **액트 4(연말)는 절대 자르지 말 것** |
