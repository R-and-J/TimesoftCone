요구사항 설계서

타임소프트콘 - 연차 경매 시스템 (B2E 연차 경매·에스크로 배당 플랫폼) · 김기철, 오지석

목적 · 요구사항을 설계 관점으로 정리한다. SRS가 “무엇을(What)”을 선언적으로 명세한다면, 본 문서는 그 요구사항을 설계 단위(도메인)로 분류·구조화하고 추적성(Traceability)을 부여하여 “어떻게 설계로 연결되는가”를 보인다. 개별 기능의 상세 동작은 기능 명세서에서 다룬다.
버전 안내 · 초기 SRS(v1.0)는 PostgreSQL·Redis 분산 락·WebSocket·MSA를 전제했고, 개발을 진행하며 학교 프로젝트 범위에 맞춰 SQLite·SQLite write 락·SSE·단일 배포 헥사고날로 고도화·확정했다. 본 문서는 “구현이 source of truth”라는 원칙에 따라 현행(as-built) 결정값을 기준으로 하며, 초기안 ↔ 현행 차이는 §9에 정리한다.

1. 문서 개요

1.1. 목적

사내 휴가 사용의 양극화 문제(고연차의 잉여 연차 소멸 ↔ 저연차의 연차 부족)를 해결하기 위한 사내 연차 경매 시스템의 요구사항을 설계 관점으로 구조화한다. 본 시스템은 근로기준법(휴식권 매매 금지)을 준수하면서 회사의 재무 리스크를 0%로 통제하는 “수익 공유 및 후 배당(Escrow & Dividend)” 아키텍처를 채택한다.

1.2. 범위

거래 대상은 특정 날짜가 아닌 “연차 사용 권리(개수)”이며, 다음 5개 하위 시스템으로 구성된다.

하위 시스템책임대응 도메인(Bounded Context)
자산화 시스템미사용 REGULAR 연차 취합 + 기여자 지분(Stake) 설정LeavePool
경매 시스템복지 콘 기반 “연차 1일권” 실시간 최고가 입찰·낙찰Auction
에스크로 시스템낙찰 대금의 중앙 수익금 대장 관리Wallet / Ledger
권한 제어 시스템낙찰 시 연차 즉시 증액 + 속성(Flag) 분리 차감 우선순위Leave
수익 배당 시스템연말 에스크로 수익을 Stake 비례로 복지카드 한도 정산Dividend
[제외 범위] 직원 간 직접 거래(P2P)와 실제 현금(KRW) 결제는 법적 규제로 영구 배제한다 - API·UI·관리도구 어디에도 P2P 경로를 두지 않는다.

1.3. 용어

핵심 용어는 용어집을 정본으로 하며, 본 문서에서 반복되는 최소 집합만 재기술한다.

용어정의
콘 (Cone)본 시스템의 입찰 재화. 복지 포인트의 도메인 명칭(VO Cone)
연차 1일권날짜에 종속되지 않는 “연차 잔여 +1” 단일 경매 매물
에스크로입찰 대금이 누적되는 중앙 수익 풀. 별도 테이블이 아니라 LEDGER_ENTRY에서 통화별 파생 집계
지분 (Stake)판매자가 공용 풀에 기여한 연차 일수 비율. 배당 정산 기준
Leave TypeREGULAR(법정/수당 대상/풀 전환) · AUCTION(낙찰/소멸/최우선 차감) · EVENT(포상/소멸)

2. 시스템 개요 및 제품 관점

2.1. 제품 관점 - 자원 소유 모델

본 시스템은 사내 그룹웨어와 독립적으로 동작하되, 핵심 자원(콘 잔액·휴가 잔액)을 자체 소유한다. 이 소유 구조가 “입찰·낙찰 경로에서 외부 호출 0”을 가능하게 하는 핵심 설계 결정이다.

자원마스터외부 결합근거 ADR
복지 콘 잔액 (wallet)본 시스템없음ADR-011
휴가 잔액 (leave_balance)본 시스템없음 (낙찰 시 내부 INSERT)ADR-016
사용자 인증/신원외부 IdP(ezpass)위임 또는 로컬 모드ADR-019~022
복지카드 한도 증액(배당 출금)외부 HRPayoutChannel(Outbox 경유)ADR-005·010
모든 외부 자원은 포트 인터페이스 뒤로 추상화되어, 실 그룹웨어 연동은 코어 도메인 무수정 상태에서 어댑터 교체만으로 가능하다(InternalLeaveAdapter → GroupwareLeaveAdapter). 학교 프로젝트 범위는 내부 어댑터까지.

2.2. 핵심 비즈니스 로직 - The 3-Way Win Model

행위자이득 구조
회사 / HR예산 선투입 0 → 에스크로 잔고 내에서만 배당 → 재무 리스크 0%. 비정규 연차(AUCTION·EVENT)는 수당 정산 제외로 예산 누수 없음
판매자
(고연차 성향)		촉진제도로 소멸될 0원짜리 연차를 풀에 기여 → 연말 배당금(복지 한도) 수령 → 100% 이득
구매자
(저연차 성향)		복지 콘 소모로 부족했던 연차 개수 확보 → 원하는 시기에 휴식

※ “판매자/구매자”는 행위 기반 역할이며 동일 직원이 두 역할을 겸할 수 있다. 근속연수가 시스템 권한을 결정하지 않는다.

2.3. 아키텍처 제약 (설계 불변식)

본 시스템을 일반 CRUD/경매 앱으로 다루면 아키텍처가 깨진다. 다음 불변식은 load-bearing이며 변경 제안 시 반드시 보존해야 한다.

#불변식근거

INV-23-플래그 분리 - REGULAR만 수당 대상·풀 전환. 세 속성 절대 병합 금지ADR-002
INV-3강제 차감 우선순위 - AUCTION → EVENT → REGULAR, 사용자 선택 불가ADR-003
