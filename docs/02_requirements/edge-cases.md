# 엣지 케이스 카탈로그

**상태**: ✅ v1 — 기획 결정 반영 (2026-05-14)
**관련 문서**: [SRS](SRS.md) / [business-rules.md](business-rules.md) / [ADR-018](../04_decisions/ADR-018-auction-settlement-rules.md)

> 정상 흐름(happy path)이 아닌 경계 상황의 *기획 결정*을 모은 문서. 여기 정의가 없으면 개발자가 임의로 지어내고, 그러면 핵심 불변식을 깰 수 있다.

---

## EC-1. 직원 퇴사

| 자원 | 처리 | 근거 |
|---|---|---|
| **Stake (배당 권리)** | **유지** — 연말 배당까지 지분 보존, 배당금 산정 대상에 포함 | 이미 기여한 노동의 대가. [ADR-008](../04_decisions/ADR-008-year-end-dividend.md) 연말 일괄 배당과 일관 |
| **wallet 잔액** | **소멸** | 복지 포인트는 사내 복지 제도의 일부 → 퇴사 시 복지 자격 종료가 자연스러움 |
| **미마감 입찰 (최고가 상태)** | 입찰 유효 유지 — 마감 시 정상 낙찰 처리. 단 낙찰 시 `AUCTION` 연차 부여 대상이 없음 → **유찰 처리로 전환** | 퇴사자에게 휴가를 부여할 수 없음 |
| **미마감 입찰 (밀린 상태)** | 이미 즉시 환불됨 ([business-rules.md](business-rules.md) §2.3) — wallet 소멸 규칙에 따라 환불분도 소멸 | — |
| **보유 중인 `AUCTION`/`EVENT` 연차** | 소멸 (연말 소멸 규칙 선행 적용 or 퇴사 시점 소멸) | [ADR-002](../04_decisions/ADR-002-leave-type-flag.md) |

> **구현 주의**: 퇴사자가 최고가인 채로 경매가 마감되면 "낙찰자는 있으나 휴가 부여 불가" 상태. 이 경우 차순위 낙찰이 아니라 **유찰(UNSOLD) 처리** — 차순위 승계는 복잡도만 키움.

## EC-2. 연중 입사자

- **입찰**: 가능 — 입사 시점부터 즉시 경매 참여 가능
- **연차 기여 (Stake)**: 불가 — 해당 연도 공용 풀에 기여한 `REGULAR` 연차가 없으므로 `contributed_days = 0`, Stake 없음
- 익년부터는 정상적으로 기여·배당 대상

## EC-3. 자기 기여분 되사기

- **허용** — 동일 직원이 판매자·구매자 역할을 동시 수행 가능 ([proposal](../01_planning/proposal.md) 행위 기반 역할)
- 매물에 "기여자" 꼬리표를 달아 본인 입찰을 막지 않음 — 구현 단순화
- 실질적으로 자기-손해(자기 `REGULAR` 연차를 포인트 내고 `AUCTION` 연차로 되사는 꼴, 수당 자격 상실)이므로 어뷰징 유인 낮음

## EC-4. 경매에 아무도 입찰하지 않음 (전부 유찰)

- 경매 마감 시 `winner_id == null` → `UNSOLD` 상태로 전이 ([04-state.md](../03_design/uml/04-state.md))
- 유찰 매물은 관리자가 사내 포상(`EVENT`)으로 수동 지급 가능 ([SRS](SRS.md) FR-4.2)
- 해당 매물 기여자의 Stake는 유지되지만 **배당 재원이 0** → 실수령 0원 ([ADR-001](../04_decisions/ADR-001-escrow-model.md) "유찰 리스크는 판매자에게 전가")
- 연말까지 미지급 유찰 재고는 영구 삭제 (익년 이월 금지)

## EC-5. 낙찰자가 연차를 사용하지 않음

- 낙찰받은 `AUCTION` 연차는 연말에 **포인트 환불 없이 소멸** ([ADR-002](../04_decisions/ADR-002-leave-type-flag.md))
- UI에 반드시 사전 고지 — "경매 연차는 연말 소멸, 환불 불가"
- 이미 지불한 포인트는 에스크로로 귀속되어 배당 재원이 됨 (시스템 정합성에는 영향 없음)

## EC-6. 입찰 동점 시도

- "최소 입찰 증분 = 고정 금액" 규칙상, 입찰액은 반드시 `현재가 + 증분` *이상*이어야 함 → 현재가와 *같은* 금액 입찰은 `BID_TOO_LOW`로 거부
- 따라서 "동일 금액 최고가 2명"은 발생 불가
- 단 *동시 도착* 입찰은 SQLite write 락(lockAuction)으로 직렬화 — 먼저 락을 잡은 입찰이 현재가를 올리고, 다음 입찰은 갱신된 현재가 기준으로 재검증

## EC-7. Stake 1위 동률 (배당 나머지 귀속)

- 배당 나머지(remainder)는 "지분 1위"에게 몰아주는데([business-rules.md](business-rules.md) §2.2), 1위가 2명 이상 동률일 수 있음
- **타이브레이크**: `user_id` 오름차순 1명에게 귀속 (결정적·재현 가능)
- 나머지는 보통 수~수십 포인트 수준이라 형평성 영향 미미. 단 규칙은 명문화 필요

## EC-8. 경매 마감 직전 입찰 폭주

- SQLite write 락(`lockAuction` no-op UPDATE)으로 입찰 직렬화 (CUT-1)
- 동시 입찰은 앞 입찰 커밋까지 대기 후 갱신된 현재가로 재검증
- 마감 시각(`end_time`) 경과 후 도착한 입찰은 `AUCTION_CLOSED` 거부 — 락 획득 여부와 무관하게 시각 우선

## EC-9. wallet 잔액 부족 상태에서 입찰

- 입찰 시 즉시 차감 모델 → 입찰 시점에 `wallet.balance >= amount` 검증 (DB-RULE-3)
- 미달 시 `POINT_INSUFFICIENT` 거부, 차감 없음
- 패자 즉시 환불 구조이므로, 다른 경매에서 밀려 환불받으면 그 즉시 재입찰 가능 (유동성 잠금 없음)

## EC-10. 연말 배치 중단·재실행

- 풀 수집·Stake 기록·배당 배치는 **재진입 가능(idempotent)** 해야 함 ([ADR-017](../04_decisions/ADR-017-leave-pool-context.md))
- 중복 실행 시 매물·Stake·배당 중복 생성 금지 — UNIQUE 제약 + `batch_run` 상태 테이블
- 배당 산정액 불일치 시 **배치 중단 + 수동 감사 대기** ([SRS](SRS.md) FR-4.1)
