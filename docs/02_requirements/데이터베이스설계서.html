데이터베이스 설계서

타임소프트콘 - 연차 경매 시스템 (B2E 연차 경매·에스크로 배당 플랫폼) · 김기철, 오지석

목적 · 본 시스템의 물리 데이터 모델을 정의한다. 엔티티 19개, 관계, 무결성 제약(DB-RULE), 그리고 SQLite 기반 as-built 특이사항을 다룬다. 본 문서는 실제 backend/prisma/schema.prisma + 마이그레이션과 1:1 동기화된 ERD 문서를 설계서 형식으로 정리한 것이며, 코드가 source of truth다.
DB 엔진 · SQLite · 파일 기반(외부 서버·컨테이너 불요). Prisma SQLite 커넥터가 enum 블록을 지원하지 않으므로, 논리적 enum(role/currency/action_type/status/leave_type)은 String 컬럼 + DB-level CHECK(… IN (…)) 로 표현된다. 아래 표의 enum 표기는 모두 "String + CHECK"로 읽는다.

3. 핵심 엔티티 상세

3.1 USER (users)

사내 직원/관리자. 신원·조직(이름/부서/직급/직책/role)은 ezpass에서 동기화. 잔액·연차 컬럼은 보유하지 않음(각각 wallet/leave_balance로 분리).

컬럼타입/제약설명
idbigint, PK내부 키 (INTEGER autoinc)
emp_idtext, UK, NOT NULL사번
nametext, NOT NULL이름
team / job_rank / job_titletext, nullableezpass 동기화 표시용
emailtext, UK, nullableezpass 로그인 키
roletext(enum), CHECKIN (ADMIN,EZPASS_ADMIN,EXAM_ADMIN,EZPASS,EXAM)
company_idbigint, FK, nullablenull = super ADMIN
password_hashtext, nullablelocal 인증(bcrypt, ADR-022)
activeboolean, DEFAULT true비활성 시 로그인 거부

3.2 WALLET (wallet) — 콘 잔액 마스터

컬럼타입/제약설명
idbigint, PKsurrogate PK (복합 PK 아님)
user_idbigint, FK→users
currencytext, CHECK IN (WELFARE_POINT)화폐 코드
balancebigint, CHECK ≥ 0, DEFAULT 0잔액

UNIQUE (user_id, currency). WelfarePointProvider(ADR-010 BiddingCurrency 구현체)만 조작 — 코어 도메인 직접 접근 금지.

3.3 LEAVE_BALANCE (leave_balance) — 연차 잔액 마스터

컬럼타입/제약설명
user_idbigint, FK
yearint, NOT NULL귀속 연도
leave_typetext(enum), CHECKIN (REGULAR,AUCTION,EVENT) — ADR-002 3-flag
granted_daysint, CHECK ≥ 0자동 부여분
adjusted_daysint, CHECK ≥ 0조정분 — 경매 낙찰 적립분
used_daysint, CHECK ≥ 0사용분

잔여 = granted_days + adjusted_days − used_days (앱 레이어 계산). UNIQUE (user_id, year, leave_type) → 연도 파티셔닝(Soft Delete 컬럼 없음).

3.4 AUCTION (auction) — 매물

컬럼타입/제약설명
idtext, PK사람이 읽는 ID (예 A-2026-106)
statustext(enum), CHECK, DEFAULT CREATEDIN (DRAFT,CREATED,OPEN,AWARDED,UNSOLD) — CUT-3 status+guard
start_price / highestbigint, CHECK ≥ 0시작가 / 현재 최고가
highest_bidder_idbigint, FK→users, nullable현재 최고 입찰자(=정산 시 낙찰자)
min_incrementbigint, CHECK > 0, DEFAULT 100최소 입찰 단위
leave_daysint, DEFAULT 1낙찰 시 부여 AUCTION 연차 일수
started_at / ends_atdatetime, CHECK ends_at > started_at오픈/마감 시각

year 컬럼 없음(연도는 id/started_at에서 파생). 인덱스 (status, ends_at), (company_id, status).

3.5 LEDGER_ENTRY (ledger_entry) — Insert-Only 원장

컬럼타입/제약설명
occurred_atdatetime, DEFAULT now()발생 시각
user_idbigint, FK
currencytext, CHECK IN (WELFARE_POINT)
action_typetext(enum), CHECK(10종)BID,REFUND,WIN,DIVIDEND,CREDIT_ADMIN,EXPIRE,REDEEM,REDEEM_REFUND,CHARGE_REQUESTED,CHARGE_REJECTED
amountbigint, NOT NULL(+) 입금 / (−) 출금
balance_afterbigint, CHECK ≥ 0기록 시점 개인 지갑 잔액
ref_notetext, CREDIT_ADMIN일 때 NOT NULL+len>0사유(감사)

트리거 ledger_entry_no_update / ledger_entry_no_delete 가 UPDATE/DELETE를 ABORT(DB-RULE-1). 에스크로 잔액은 이 테이블 합계로 파생(DB-RULE-4).

3.6 STAKE (stake) — 연도별 배당 지분

LeavePool 수집이 (user_id, year)당 1행 적재 → 배당이 해당 연도 stake를 읽어 분배. days(기여 REGULAR 일수)만 저장하고 stake_ratio는 앱 레이어에서 해당 연도 합 대비 동적 계산. UNIQUE (user_id, year).

3.7 풀·교환·충전 워크플로 테이블

연말 자산화(LeavePool)와 자립형 배포(교환·충전)를 지원하는 보조 테이블. 멱등 마커는 UNIQUE로 중복 실행을 차단한다.

테이블핵심 컬럼 · 제약
leave_pool_run연말 수집 멱등/감사 마커. source_year/target_year(=source+1)/집계. UNIQUE (company_id, target_year)
leave_pool_supply풀 재고. contributed_days(불변)/remaining_days(미발행). UNIQUE (company_id, target_year, user_id)
release_policy발행 분산 정책(싱글톤 id=1, CHECK). cadence IN (none,daily,weekly,monthly)
outbox트랜잭션 아웃박스. topic(예 HR_LEAVE_GRANT)/status(PENDING/SENT/DEAD)/지수 백오프 재시도
redemption_request교환 신청→승인→수령. status IN (PENDING,APPROVED,RECEIVED,REJECTED), 가격 스냅샷, coupon_code
charge_request충전 요청→승인. amount(CHECK>0)/status IN (PENDING,APPROVED,REJECTED). 승인 시 같은 tx에서 CREDIT_ADMIN 적립

6. as-built 특이사항

항목내용
enum → String + CHECKSQLite/Prisma가 enum 블록 미지원 → 모든 논리 enum은 String + CHECK(… IN (…))
PK 타입Prisma상 BigInt autoincrement지만 SQLite는 INTEGER PK AUTOINCREMENT로 채번. 일부 후발 테이블(stake/leave_pool_*/outbox/redemption_*/charge_request/release_policy)은 Int PK로 강등
ESCROW 테이블 없음에스크로 잔액은 ledger_entry 합계 파생 → v2 설계의 ESCROW 엔티티/CHECK 제거
AUCTION_BATCH 없음그 역할은 leave_pool_run(연말 풀 수집 멱등 마커)이 수행
AUCTION PK = String디자인 목업 ID(A-2026-106)와 1:1 매칭 위해 문자열 PK 채택
FK 부여 위치FK는 마이그레이션 레벨에서 부여(스키마는 스칼라만 — 백릴레이션 클러터 회피)
