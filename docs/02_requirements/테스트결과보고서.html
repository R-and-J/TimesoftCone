1. 테스트 환경 · 전략

구분내용
테스트 러너Jest (+ ts-jest 트랜스폼), testEnvironment: node
단위 테스트도메인 순수 로직 (DB 없음). *.spec.ts — 소스 옆에 배치, rootDir=src

스위트결과검증 내용 (대표 케이스)
auction.spec.tsPASS생성/오픈/입찰/정산 상태 전이, 최소 증분 검증, 직전 리더 반환(환불용), 마감 후 입찰 거부, 자기 재입찰 거부, anti-snipe 연장(CUT-5)
wallet.spec.tsPASS0 잔액 개시, 적립/차감, 잔액 부족 시 InsufficientConeError
leave-deduction.spec.tsPASSFR-3.1 우선순위 — AUCTION→EVENT→REGULAR 순차 소진, 빈 버킷 스킵, 잔여 부족 예외, 비양수 일수 거부
leave-pool-plan.spec.tsPASS기여자별 stake 기록(userId 정렬, 0일 제외), 기여 없을 때 빈 계획
release-plan.spec.tsPASS분산 발행 — daily/weekly/monthly 스케줄 계산, 잘못된 형식·quantity=0 예외, none=즉시 전량
cone.spec.tsPASSVO Cone — 음수/비정수 거부, add/subtract 불변성, 잔액 부족 예외
auction-id.spec.tsPASSVO AuctionId — A-YYYY-NNN 형식 검증, 값 동등성
user-id.spec.tsPASSVO UserId — 양의 정수만 허용, 값 동등성

3. 통합 테스트 (e2e, 실제 SQLite)

5 suites · 24 tests · 전부 통과 (2.48s) — 실제 DB 트랜잭션으로 핵심 시나리오를 단언.

스위트결과검증 내용
place-bid.e2e-spec.tsPASS입찰자 차감 + BID 원장 1건을 단일 tx로, 직전 리더 자동 환불(원래 BID는 Insert-Only 보존), 잔액 부족 시 전체 롤백, anti-snipe 마감 연장 영속화
settle-dividend.e2e-spec.tsPASS정산(AWARD + AUCTION 휴가 부여) 후 연말 배당 Σ배당 = 에스크로(NFR-2), 재정산 시 멱등 거부(이중 지급 없음)
leave-pool.e2e-spec.tsPASSREGULAR 미사용 → stake+supply 수집, 동일 연도 재수집 409 멱등 거부, dryRun 미기록 미리보기, cadence=none 일괄 발행, supply 빈 경우 EMPTY
leave-admin.e2e-spec.tsPASS휴가 차감 우선순위(ADR-003), 잔여 부족 시 409(행 변경 없음), UNSOLD→EVENT 전환, 비-UNSOLD 전환 거부, 연도별 유찰 청산 멱등
admin-and-activity.e2e-spec.tsPASSFR-5.1 관리자 적립(+CREDIT_ADMIN 원장·사유 필수), 빈 사유 거부, CREDIT_ADMIN 에스크로 제외(NFR-2), 활동 조회 정렬·요약, 무입찰 UNSOLD 정산
