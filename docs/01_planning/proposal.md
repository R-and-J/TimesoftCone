# 프로젝트 기획안 — 사내 연차 경매 시스템

**팀명**: 타임소프트콘 | **팀원**: 김기철, 오지석

---

## 1. 프로젝트명

**연차 경매 시스템** (Annual Leave Auction System)

## 2. 기획 배경 및 목적

### 문제 인식

| 집단 | 상황 |
|---|---|
| **고연차 직원** | 업무 과다로 연차를 다 쓰지 못함 → 연말에 **미사용 연차가 소멸**되거나 수당으로만 정산 |
| **저연차 직원** | 법정 연차가 적어 **휴가 부족** 호소 |

양쪽 모두에게 손해인 **연차 양극화 현상**이 사내에서 반복적으로 발생.

### 목적

남는 휴가와 필요한 휴가를 **경매 방식으로 매칭**함으로써 사내 복지의 유연성을 극대화한다.

## 3. 기초 아이디어 (검토 후 폐기)

> "직원 간 경매를 통해 연차를 직접 사고팔 수 있도록 한다."

### ❌ 폐기 사유: 법과 제도의 벽

- **근로기준법 제60·61조**: 연차 유급휴가는 근로자의 **휴식권**이며, 이를 **매매하는 행위는 금지**됨.
- 개인 간 직접 거래(P2P) 및 현금 결제 구조는 **위법 소지**가 명백하므로 시스템 설계에서 **영구 배제**.

## 4. 보완된 방안 — B2E Escrow & Dividend 아키텍처

### 핵심 전환

| 전환 축 | 초기 아이디어 | 최종 설계 |
|---|---|---|
| 거래 주체 | 직원 ↔ 직원 (P2P) | **회사가 중개** (B2E) |
| 재화 | 현금(KRW) | **사내 복지 포인트** |
| 정산 시점 | 즉시 지급 | **연말 일괄 배당** |
| 매물 단위 | 날짜 종속 연차 | **"연차 1일권"** (날짜 비종속 개수) |

### 5개 하위 시스템

1. **자산화 시스템** — 미사용 연차 취합 및 기여자 지분(Stake) 설정 (연말 배치)
2. **경매 시스템** — 복지 포인트 기반 실시간 최고가 입찰·낙찰
3. **에스크로 시스템** — 낙찰 대금을 임시 보관하는 중앙 수익금 대장
4. **권한 제어 시스템** — 낙찰 시 HR API 연동으로 연차 개수 증액 + 속성(Flag) 분리
5. **수익 배당 시스템** — 연말에 에스크로 수익금을 지분율대로 복지카드 한도로 정산

## 5. 3-Way Win Model (참여자별 가치)

| 참여자 | 얻는 가치 |
|---|---|
| **회사 / HR** | 예산 선투입 0원, 재무 리스크 0%. 비정규 연차(AUCTION/EVENT)는 수당 정산에서 배제되어 예산 누수 없음 |
| **판매자 / 고연차** | 소멸될 뻔한 0원짜리 연차 → 연말 배당금(복지 한도) 수령 |
| **구매자 / 저연차** | 포인트 소모로 부족한 연차 개수 확보, 원하는 시기에 휴식 |

## 6. 시스템 경계 (Scope)

### 포함
- SSO 인증 기반 사내 직원 대상 경매·입찰·낙찰
- HR API 연동 (연차 부여 / 복지카드 한도 증액)
- 에스크로 원장 관리 및 연말 배당 배치
- 관리자 유찰 재고 수동 지급

### 제외 범위
- **직원 간 직접 거래 (P2P)** — 법적 제약으로 영구 배제
- **실제 현금(KRW) 결제** — 복지 포인트만 사용
- **사외 연차·휴직·육아휴직** — 일반 연차(`REGULAR`)만 대상

## 7. 핵심 성공 지표 (KPI 초안)

- 연말 에스크로 잔액 = 포인트 로그 총합 (재무 정합성 100%)
- 동시 입찰 Race Condition 0건 (NFR-1)
- 연말 배당금 오차 0원 (NFR-2)
- 연차 소멸률 감소 (대비 기준 연도 대비)
