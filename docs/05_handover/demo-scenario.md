# 시연 시나리오 문서 (제출용)

> **대상 시스템**: 타임소프트콘 - B2E 연차 경매·에스크로 배당 플랫폼
> **목적**: 라이브 시연의 **순서**, 단계별 **예상 입력값**, **기대 결과**를 사전 정의하고,
> 시연 실패 시 사용할 **대체 자료(스크린샷)**를 규정한다.

---

## 1. 시연 순서 · 예상 입력값 · 기대 결과

전체 5막. 각 막의 **단계 / 조작·입력값 / 기대 결과**를 표로 정리한다.

### 액트 1 - 사용자가 1일권을 산다 (입찰 → 자동 환불 → 낙찰 → 연차 부여)

| # | 단계 | 예상 입력값 (조작) | 기대 결과 |
|---|---|---|---|
| 1-1 | 로그인 + 대시보드 | 창 A: `exam001@exam.com` / `1234` | 우상단 잔액 **100,000 콘**, 내 연차 위젯 **REGULAR 15일**(파란 막대) 표시. `@exam.com` → 로컬 인증 자동 라우팅 |
| 1-2 | 경매 목록 → 상세 → 입찰 | 경매 목록 → Grid 뷰 → 1일권 매물 클릭 → **30,100 콘** 입찰 | 시작가 30,000 콘 표시, 잔액 즉시 **−30,100**(= 69,900 콘). 지갑 차감+에스크로+원장 INSERT 단일 트랜잭션 |
| 1-3 | 다른 사용자 덮어쓰기 → 자동 환불 | 창 B: `exam002@exam.com` 로그인 → 같은 매물 **30,200 콘** 입찰 | 창 B 잔액 **−30,200**, "최고가 입찰" 토스트. 창 A 새로고침 시 잔액 **+30,100 복구**(100,000) + "덮어쓰기 당했습니다" 알림 |
| 1-4 | 즉시 마감 → 낙찰 → AUCTION 연차 | 관리자: `super@admin.local` → `/admin/auctions` → 해당 매물 **즉시 마감** | 창 B: "낙찰! AUCTION 1일 부여됨" 토스트, 대시보드에 **AUCTION 1일(노란 막대)** 추가. 잔액 변동 없음(30,200은 에스크로 확정). 외부 호출 없는 단일 DB 트랜잭션 |

### 액트 2 - 스쿱 마켓 → 수령 대기 → 보관함

| # | 단계 | 예상 입력값 (조작) | 기대 결과 |
|---|---|---|---|
| 2-1 | 교환 신청 | 창 A `user001`(김민준, ezpass) → 스쿱 마켓 → **AI 구독권 (1개월)** → 신청 | 잔액 **−30,000 콘** 잠금, "교환 신청 완료" 토스트 |
| 2-2 | 관리자 승인 + 안내문 발급 | 관리자 → 「교환 신청」 탭 → 「대기」 KPI 클릭(필터) → 승인 → 안내문(쿠폰코드 등) 입력 | 신청 상태 **승인**으로 변경, 안내문 저장. KPI 카드 클릭=필터, 재클릭=해제 |
| 2-3 | 수령 대기 → 수령 완료 → 보관함 | 창 A 새로고침 → 「📦 수령 대기 1건」 카드의 **수령 완료** 클릭 → 「보관함(N)」 클릭 | 우상단에 안내문 포함 수령 대기 카드 노출, "수령 완료" 시 카드 사라짐, 보관함 모달에서 안내문 **영구 재확인** |

### 액트 3 - 관리자 운영 한 바퀴

| # | 단계 | 예상 입력값 (조작) | 기대 결과 |
|---|---|---|---|
| 3-1 | 운영 KPI + 내보내기 | `super@admin.local` → `/admin/ops` → "정산 데이터 내보내기" | KPI 5칸(에스크로·진행·예정·유찰·오늘 낙찰) 표시, Excel/MD/JSON 파일 다운로드 |
| 3-2 | 유찰 매물 처리 ① EVENT | "유찰 재고" 카드 → 검색 `exam003` → 클릭 → **EVENT 휴가 지급** | "exam003에게 EVENT 1일 지급" 토스트, KPI 유찰 **−1** |
| 3-2 | 유찰 매물 처리 ② REOPEN | 다른 유찰 매물 → **REOPEN** 토글 → 마감 시각 입력 → 재경매 | "기존 매물 → 새 매물 ID 재오픈" 토스트, `/admin/auctions`에 새 매물 노출 |
| 3-3 | 스토어 상품 관리 | 「스토어」 탭 → 「상품 추가」 → SKU `DEMO-001`, 이름 "데모 상품", 가격 **5,000 콘** → 저장 → 비활성화 토글 | 표에 행 즉시 추가 + 감사 로그 **CREATE**, 비활성화 시 **DEACTIVATE** 로그. 창 A `/redemption` 새로고침 시 비활성 상품 **카탈로그에서 사라짐** |
| 3-4 | 회원관리 두 탭 | `/admin/members` → EZPASS 탭 ↔ EXAM 탭 전환 | EZPASS 탭: 회원 추가·작업 컬럼 **없음**, 「지금 동기화」만. EXAM 탭: 「회원 추가(exam)」·행별 수정/비활성 **있음** |
| 3-5 | 충전 처리 (옵션) | EZPASS 회원 이서연(T002) 행 "콘 관리" → **+150,000 콘** / 사유 "우수 사원" → 적용 | "이서연 충전 — +150,000 콘 (잔액 153,600 콘)" 토스트, 원장 **CREDIT_ADMIN** INSERT (사유 필수) |

### 액트 4 - 12/31의 마법 (핵심: 풀 수집 + 배당 → 에스크로 0)

| # | 단계 | 예상 입력값 (조작) | 기대 결과 |
|---|---|---|---|
| 4-1 | 사전 설명 | (멘트) 시드에 작년(2025) 39건 + stake 10명 존재 | - |
| 4-2 | 연말 풀 수집 | `/admin/ops` → "풀 수집" → **dryRun** → "수집 실행" | 기여자별 일수 미리보기(예: 기여자 39명·623일·**2027 매물 623개**), 익년도 1일권 매물 생성(`LP-2027-*`), `/admin/auctions`에 노출. `leave_pool_run.targetYear` UNIQUE로 연도당 1회 |
| 4-3 | 연말 배당 정산 | "연말 배당" → **dryRun** 미리보기 → **실지급** | dryRun: 김민준 **1위 23.4%(25일)** / 동률 **7.5%(8일)** 다수 / **3.7%(4일)** 등 25명, 합계 = 에스크로 정확히 일치. 실지급 후: "25명에게 N콘 배당" 토스트, **에스크로 → 0**, 원장에 DIVIDEND 25건 INSERT |
| 4-4 | 멱등성 확인 | "연말 배당" **한 번 더** 실행 | **HTTP 409**("이미 배당이 정산되었습니다"). 회사 스코프의 LedgerEntry(actionType=DIVIDEND) 존재로 멱등 차단 |

### 액트 5 - 회사별 완전 독립 (멀티테넌트)

| # | 단계 | 예상 입력값 (조작) | 기대 결과 |
|---|---|---|---|
| 5-1 | 회사 스위처 | `super@admin.local` → 우상단 스위처 → EZPASS ↔ EXAM 전환 | 회원·매물·원장·통계가 회사별로 **완전히 교체** |
| 5-2 | 다른 회사로 직접 로그인 | 새 시크릿창 → `examadmin@exam.com` → `/admin/members` | EXAM 직원만 표시, EZPASS 데이터 없음, **회사 스위처 미노출**. JWT `companyId`로 모든 쿼리 자동 스코프 |
| 5-3 | 새 회사 추가 (멘트만) | (구두) `company` 행 + 사용자 `companyId` 추가 | 코드 변경 0으로 신규 회사 온보딩 가능, 도메인 추가 시 인증 라우팅 자동 |
