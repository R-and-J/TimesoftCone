# ADR-008: 연말 일괄 배당 (즉시 분배 불가)

- **상태**: 🟡 Proposed (ADR-001의 필연적 결과)
- **결정일**: _미결_
- **결정자**: 타임소프트콘

## 컨텍스트

[ADR-001](ADR-001-escrow-model.md)에서 "낙찰 대금을 에스크로에 누적 → 연말 배당" 모델을 채택했다. 그렇다면 **왜 즉시 배당하지 않고 연말까지 모아두는가?**를 명문화할 필요가 있다.

## 초기 고민 (Rejected)

> **옵션 — 즉시 분배**: "매 낙찰 시점에 해당 매물의 기여자에게 콘을 즉시 지급한다."

예: 매물 A가 10,000 콘에 낙찰되면, 그 매물을 만든 기여자 B에게 즉시 10,000 콘 지급.

## 회피한 리스크

💣 **리스크 1: 매물-기여자 1:1 매핑 붕괴**

공용 풀 모델에서는 "매물 A"가 특정 개인의 연차가 아닌 **전체 반납 연차의 1개**. 즉시 분배하려면:
- 매물마다 기여자를 1:1로 고정 → 공용 풀 철학 훼손
- 아니면 낙찰 시점마다 당시 지분율로 즉시 N분 → **연중 지분율이 계속 바뀜**(후속 반납자 등장)

💣 **리스크 2: 지분 계산의 시점 불일치**

- 1월 낙찰분은 1월 기준 지분으로 분배
- 12월 낙찰분은 12월 기준 지분으로 분배
- → **같은 1콘 낙찰이 시기에 따라 다른 가치**를 지니게 되어 **공정성 훼손**

💣 **리스크 3: 유찰 처리 불확실성**

- 즉시 분배를 하면 이미 분배된 매물의 유찰 상황이 없지만, 유찰 재고(`FR-4.2`)는 다음 연도 이월 불가 → 연중 어느 시점에 "이 매물은 최종 유찰"이라고 확정 가능한가?
- 결국 연말까지 기다려야 최종 상태가 결정됨

## 결정

**연말 일괄 배당 (Year-End Lump-Sum Distribution)**

- 모든 낙찰 수익은 `ESCROW` 원장에 연중 누적만
- 12월 31일 배치에서:
  1. 연간 총 에스크로 잔액 확정
  2. 각 기여자의 최종 `stake_ratio` 산정 (`contributed_days / total_contributed_days`)
  3. 배당금 = 에스크로 총액 × stake_ratio
  4. HR API `/api/hr/welfare`로 복지카드 한도 증액 호출

## 결과 및 트레이드오프

### ✅ 긍정적 결과
- **공정성 보장** — 모든 기여자가 동일 기준(연말)으로 분배받음
- **시스템 단순성** — 연중 실시간 분배 로직 불필요
- **유찰 재고 처리 통합** — 연말 시점에 전체 상태를 한 번에 결정
- **감사 용이성** — 배치 1회로 검증 가능

### ⚠️ 트레이드오프
- **판매자 체감 보상 지연** — 1월에 연차 반납한 직원은 **거의 1년 뒤**에 배당 수령 → 참여 유인 저하 가능
- **배치 1회 실패 = 전체 연도 영향** — 12/31 배치의 무결성이 절대적으로 중요
- **예상 배당금 가변성** — 연중 대시보드에 "예상 배당"을 표시하더라도 연말까지 변동

### 🛡️ 제약
- **[FR-4.1]** 에스크로 잔액 초과 배당 절대 불가
- **[SRS UI-1]** 연중 대시보드에 **"예상"**임을 명확히 표시 (최종 아님)
- **배치 실패 대응 계획 필수** — 부분 실패 시 이어서 재개 가능한 재진입성(idempotent) 설계
