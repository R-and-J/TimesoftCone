# ADR-009: 기존 복지 콘 재활용 (신규 화폐 미발행)

- **상태**: ✅ Accepted (개정 v2)
- **결정일**: 2026-04-10 (초안) → 2026-05-14 (개정)
- **결정자**: 타임소프트콘
- **개정 사유**: 본 ADR의 *정책* 결정과 *구조* 결정을 분리. 구조 측면은 [[ADR-010]] (Currency 추상화)로 위임. 전제 조건 충족 불가 상황은 [[ADR-011]] (복지 콘 시스템 자체 보유)로 보완.

## 컨텍스트

경매의 재화(결제 수단)로 무엇을 사용할지 결정 필요.

## 후보 옵션

### 옵션 A — 기존 사내 복지 콘 재활용 (권장)
### 옵션 B — 경매 전용 신규 화폐(예: "LEAVE COIN") 발행
### 옵션 C — 현금(KRW) 결제

## 회피한 리스크

### 옵션 C 회피
💣 **근로기준법 위반** — 휴식권 매매 금지 원칙. [SRS 1.1](../02_requirements/SRS.md#11-목적) 및 [proposal.md](../01_planning/proposal.md)에서 이미 배제.

### 옵션 B 회피
💣 **"유가증권" 취급 리스크**
- 사내 전용이더라도 새 화폐를 발행하면 **세무적으로 현금성 자산**으로 분류될 수 있음
- 충전/환불 정책 수립 필요 → 금융 당국 심사 가능성
- 직원 관점에서 **"또 하나의 사내 콘"** → 인지 부하

💣 **시스템 부담 증가**
- 화폐 발행·회수·만료 정책 별도 설계
- 기존 복지 콘과의 환율·변환 규칙

## 결정

**옵션 A — 사내 기존 복지 콘(Welfare Points) 재활용**

- 직원이 이미 보유한 복지 콘으로 입찰·낙찰
- 연말 배당금도 복지카드 한도 증액 형태 (동일 화폐계)
- 신규 화폐 미발행 → 시스템 및 세무 단순화

### 본 결정의 범위 (개정 v2)

본 ADR은 **"현재 어떤 화폐를 사용할지"의 *정책* 결정**이다. 다음은 본 ADR의 범위가 아니다:

- 화폐를 *어떻게 결합할지* (구조) → [[ADR-010]] 통화 추상화
- 잔액 마스터를 *누가 보유할지* (책임) → [[ADR-011]] 본 시스템이 wallet 마스터 보유

본 ADR은 [[ADR-010]]의 *첫 번째 구현체* `WelfarePointProvider`를 정의하는 *정책*이다.
미래에 다른 화폐로 교체되면 본 ADR이 *Superseded* 처리되고, 코어 도메인은 변경되지 않는다.

### 전제 조건 (개정 v2 — 조건부)

본 ADR은 다음 중 *하나의* 조건이 충족될 때 유효하다:

1. **(외부 시스템 시나리오)** 회사에 이미 운영 중인 복지 콘 시스템이 존재하고, `복지 콘 → 복지카드 한도` 변환 API(HR `/welfare`)가 제공되는 경우 → 본 시스템은 *어댑터* 역할
2. **(자체 보유 시나리오 — [[ADR-011]])** 외부 시스템이 부재한 경우, 본 시스템이 잔액 마스터를 직접 보유 → 본 시스템이 *마스터*. 배당 시에만 HR `/welfare`로 한도 통보

**현재 본 프로젝트는 시나리오 2를 채택** — [[ADR-011]] 참조.

## 결과 및 트레이드오프

### ✅ 긍정적 결과
- **법·세무 리스크 최소** — 기존 사내 제도 내에서 순환
- **직원 인지 부하 낮음** — 이미 쓰고 있는 콘 재활용
- **개발 스코프 축소** — 화폐 발행/환불 로직 불필요
- **단일 화폐계** — 입찰·배당·수당이 모두 같은 단위로 표현되어 직관적

### ⚠️ 트레이드오프
- **복지 콘 고갈 시 입찰 불가** — 복지 콘을 경매에 소진한 직원은 기존 복지(헬스장 등) 이용에 제약
- **기존 시스템 의존** — 복지 콘 시스템 변경 시 본 시스템에도 영향
- **콘 적립 경로 제한** — 본 시스템에서 추가 콘을 만들 수 없음 (수혜자는 기존 적립 경로를 통해서만 충전 가능)

### 🛡️ 제약
- 복지 콘의 **음수 잔액 절대 불가** (`wallet.balance >= 0` CHECK)
- 입찰 시 콘은 **홀드(hold)** 개념 없이 **즉시 차감** (단순성 우선). 유찰/입찰 취소 시 REFUND 로그로 환불.
- 복지 콘 정책 변경(유효기간·환율) 시 본 시스템 영향도 분석 필수
